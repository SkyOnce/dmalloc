The following set of tasks should be performed when the dmalloc library gets released.

1) Update the version number in version.h and where else it lists.
2) Update the ChangeLog files.
3) Update release number in version.h and the files listed there.
4) Update release number and date in dmalloc.texi and build doc files.
5) Run autoconf to make sure configure up to date.
6) Check in all files.
7) git tag the release with dmalloc_release_X_X_X
8) make distclean ; autoconf ; configure ; make release
9) Run tests:
	- [ maybe mkdir x ; cd x ]
	- ./configure or ../configure
	- make all heavy
	- maybe install the new dmalloc utility to test
	- dmalloc -l malloc high
	- make heavy
	- examine the log file, check version, look for exceptions
	- [ maybe dmalloc_t -r ] to check for wierd flag conditions

10) Install the tgz file and html file in the dmalloc.com repo
11) Edit releases/index.html to include the new version
12) Installed the updated doc files into the dmalloc.com repo.
